\abstracten

Structuring course knowledge can support teaching and research in computer networking. However, course documents contain specialized terms and complex relations, while manual knowledge graph construction is costly and hard to scale. This work studies how to build knowledge graphs automatically from unstructured course materials with large language models (LLMs).

A seven-stage pipeline was built that converts PDFs into Markdown, text chunks, raw triples, a merged knowledge graph, Neo4j storage, and a Next.js visualization frontend. Multimodal vision-language models perform OCR (Optical Character Recognition) and produce structured Markdown. A recursive header-aware chunking algorithm keeps document hierarchy within token limits. Chinese prompts extract four entity types (protocol, concept, mechanism, metric) and four relation types (contains, depends\_on, compared\_with, applied\_to) in zero-shot and few-shot modes. An evidence verification gate checks that each triple is supported by source text. Entity canonicalization combines FAISS (Facebook AI Similarity Search) similarity search with the Union-Find disjoint-set algorithm to merge duplicate entities.

The pipeline processed 205 computer networking course documents and generated 16,438 text chunks ($\sim$10.4 million tokens). The final knowledge graph (\texttt{merged-fs-recheck}, with LLM-based candidate review enabled) contains 32,760 entities and 17,627 relational triples across the predefined entity and relation categories. On an independent set of 500 expert-reviewed gold triples, this work reports a four-tier matching protocol (Strict / Substring / Slot / Schema-level): triple-level F1 reaches 15\%--19\% under substring matching, 37\%--42\% under slot matching, and 74\%--82\% under schema-level matching. Frequent entities such as TCP, UDP, HTTP, and IP show that the graph captures core networking concepts. The result provides a structured knowledge graph for networking education.
