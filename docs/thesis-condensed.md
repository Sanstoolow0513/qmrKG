# 基于大语言模型的课程知识图谱自动化构建（论文浓缩版）

## 一、研究背景与意义

计算机网络课程材料以 PDF / PPT 等非结构化形式为主，包含大量专业术语与跨章节复杂语义关系，人工整理结构化文本耗时长且难以维护。传统知识图谱构建依赖大量人工标注，面临成本高、领域适应性差、知识更新滞后等问题。GPT、DeepSeek 等大语言模型（LLM）借助零样本与少样本学习能力，为低标注场景下的知识抽取提供了新路径。本课题以 LLM 为核心，通过提示工程与流水线工程，将课程文档自动转化为结构化知识图谱，为教学资源的智能化利用提供可查询的知识基础。

## 二、相关工作

NER 方向：GPT-NER 借助 `@@##` 标记将 NER 转为生成任务；SLIMER 通过详尽的类型定义支持零样本识别；CodeIE / GoLLIE 用代码形式表达抽取任务以提升结构化输出能力。RE 方向：GPT-RE 提出任务感知示例检索；AutoRE 提出 Relation-Head-Fact 分解范式；Revisiting-RE 引入思维链辅助文档级关系判断。统一抽取与知识图谱方向：InstructUIE 通过指令微调统一多任务；OneKE 采用多智能体协作；EDC、KGGen、SAC-KG 在实体规范化与质量控制上提出代表性方案。教育领域：Educational-KG-Pipeline 与 Graphusion 给出端到端构建参考。现有工作较少将 NER、RE 与端到端流水线在课程知识场景下系统整合，且在证据可追溯性与零/少样本提示对比方面仍有空白。

## 三、系统设计

**Schema** 定义四类实体（protocol / concept / mechanism / metric）与四类关系（contains / depends\_on / compared\_with / applied\_to），关系附带头尾类型约束。

**两阶段抽取（Propose–Review）**：提议阶段由 LLM 生成候选实体与三元组，侧重召回；审核阶段独立调用 LLM 逐条验证关系类型合法性、证据是否为原文连续子串、头尾实体是否出现在证据中、是否自环、是否引入新实体。

**证据门控**：每条三元组必须附带原文连续子串作为 evidence，实现可验证、可追溯并抑制幻觉。

## 四、系统实现

![图 2：七阶段流水线总体架构（对应论文 fig:4-1）](TODO: 图片路径)

七阶段流水线：文档转图（PyMuPDF / LibreOffice，DPI 200）→ VLM OCR（Qwen3-VL-8B 输出 Markdown）→ 文本分块（tiktoken cl100k\_base，递归标题感知，T\_max=4000）→ 三元组抽取（DeepSeek-v4-flash，支持 zero-shot / few-shot 两种模式）→ 知识融合（别名 + 嵌入规范化）→ Neo4j 导入（Cypher MERGE 幂等写入）→ 前端可视化（Next.js 16 + react-force-graph-2d，诱导子图采样、对数节点映射、缩放感知标签、多边曲线偏移）。

LLM 工厂模式（TaskLLMFactory）按任务名加载模型 Profile，统一管理速率限制（滑动窗口 RPM 限流）、并发信号量与指数退避重试；各阶段以文件系统中间产物解耦，支持断点续跑。

## 五、实验与结果

语料：205 份 PDF（教材、课件、RFC、实验、试题），生成 16,438 文本块、约 1,040 万 token。

最终图谱（fs-recheck）：32,760 实体（concept 69.3% / mechanism 19.6% / metric 6.7% / protocol 4.5%），17,627 关系（contains 45.3% / applied\_to 24.5% / depends\_on 21.2% / compared\_with 9.0%），连通节点 13,018，平均度 2.71。高频实体 TCP（2,117）、UDP（1,069）、HTTP（701）、IP（640），高频关系 TCP↔UDP 比较 155 次、HTTP depends\_on TCP 58 次。

500 条独立金标四档评估（zs/fs × 有/无复核）：严格 F1 11%–16%、软匹配 15%–19%、槽位 37%–42%、类型级 74%–82%（zs-recheck 类型级 F1 达 81.93%）。阶梯式上升表明失分主要来自实体表面形式差异（中英缩写并用），而非关系类型判断错误；复核机制在严格档下略降字面命中、在槽位与类型级档下提升语义覆盖；few-shot 仅在 tail\_type 分类（89.04% vs 75.24%）局部领先。

## 六、结论与展望

本课题实现了从课程 PDF 到可交互知识图谱的端到端自动化构建，验证了 LLM 在低标注、专业领域知识抽取中的可行性。主要贡献包括：面向课程知识的四类实体与四类关系 Schema、Propose–Review 两阶段抽取与证据门控机制、别名 + 嵌入两级实体规范化、七阶段可断点续跑流水线，以及"严格／软匹配／槽位／类型级"四档评估口径。后续可在金标风格对齐的少量高质量示例、跨学科 Schema 迁移、知识图谱与 RAG 教学问答的结合等方向深入。
